# Система типов языка MiniC

MiniC — статически типизированный язык с явными объявлениями типов,
value semantics и запретом неявных приведений.

---

## 1. Общие свойства

- Типизация статическая: тип каждого выражения определяется на этапе компиляции.
- Типизация номинальная:
  - встроенные типы различаются по имени;
  - структуры различаются по полному имени типа, а не по совпадению набора полей;
  - размер массива является частью его типа.
- Неявные приведения типов запрещены.
- Все значения передаются, возвращаются и присваиваются по значению.
- Синоним типа (`type`) не создаёт новый тип, а только даёт альтернативное имя
  уже существующему типу.

---

## 2. Встроенные типы

MiniC поддерживает следующие встроенные типы:

### 2.1 Целочисленные знаковые

- `int8`
- `int16`
- `int32`
- `int64`

### 2.2 Целочисленные беззнаковые

- `uint8`
- `uint16`
- `uint32`
- `uint64`

### 2.3 Вещественные

- `float32`
- `float64`

### 2.4 Логический, символьный и строковый

- `bool`
- `char`
- `string`

### 2.5 Специальный тип

- `void`

Тип `void` допустим только как тип возврата функции.
Переменные, параметры, элементы массива и поля структуры типа `void` запрещены.

---

## 3. Пользовательские и составные типы

### 3.1 Структуры

Тип, объявленный через `struct`, является пользовательским составным типом.

Пример:

```minic
struct Point {
    int32 x,
    int32 y
}
```

- Поля структуры упорядочены.
- Каждое поле имеет собственное явно указанное имя и тип.
- Поля и методы структуры могут иметь видимость `public` или `private`.
- Две структуры с одинаковыми полями, но разными именами, считаются разными типами.
- Квалифицированные имена вида `Math::Point` являются отдельными номинальными типами.
- Метод вызывается через выражение вида `value.method(...)` и семантически получает неявный `self`.

### 3.2 Массивы

Массив имеет форму `T[N]`, где:

- `T` — тип элемента;
- `N` — положительный целочисленный размер, известный во время компиляции.

Свойства массива:

- массив имеет фиксированный размер;
- размер входит в тип;
- `int32[3]` и `int32[4]` — разные типы;
- многомерные массивы в текущей версии языка не поддерживаются.

### 3.3 Синонимы типов

Синоним объявляется так:

```minic
type Meters = int32;
```

Синоним:

- не создаёт новый тип;
- полностью прозрачен для проверки совместимости;
- может ссылаться как на встроенный тип, так и на массив или структуру.

### 3.4 Типы из модулей

- Пользовательские типы могут приходить из импортированных модулей.
- Полное имя такого типа включает путь модуля, например `Math::Point`.

---

## 4. Равенство и совместимость типов

Два типа считаются одинаковыми, если выполняется одно из правил:

- встроенные типы имеют одинаковое имя;
- массивы имеют одинаковый тип элемента и одинаковый размер;
- структуры имеют одно и то же полное имя;
- один или оба типа являются alias, раскрывающимися в один и тот же базовый тип.

Совместимость типов в MiniC строгая:

- присваивание допустимо только при полном совпадении типов;
- аргумент функции должен точно совпадать с типом параметра;
- тип возвращаемого выражения должен точно совпадать с типом возврата функции;
- инициализатор переменной должен совпадать с объявленным типом переменной.

Единственное исключение — контекстная типизация числовых литералов, описанная ниже.

---

## 5. Типы литералов

### 5.1 Булевы, символьные и строковые литералы

- `true` и `false` имеют тип `bool`
- символьный литерал имеет тип `char`
- строковый литерал имеет тип `string`

### 5.2 Целочисленные литералы

`INT_LITERAL` семантически трактуется как целочисленный литерал.

Поддерживаются десятичные, шестнадцатеричные (`0x...`) и двоичные (`0b...`) формы.

Правила:

- если контекст ожидает конкретный целочисленный тип, литерал получает этот тип,
  если значение представимо в нём;
- если контекст не задаёт тип, литерал получает тип `int32`.

Примеры:

```minic
let int64 x = 1;   // литерал 1 трактуется как int64
let int32 y = 1;   // литерал 1 трактуется как int32
```

### 5.3 Вещественные литералы

`FLOAT_LITERAL` семантически трактуется как вещественный литерал.

Поддерживаются:

- обычная дробная запись;
- экспоненциальная запись, например `1.25e2`;
- специальные значения `inf`, `-inf` и `NaN`.

Правила:

- если контекст ожидает `float32` или `float64`, литерал получает ожидаемый тип,
  если значение представимо в нём;
- если контекст не задаёт тип, литерал получает тип `float64`.

Эти правила относятся к типизации литералов и не считаются неявным приведением.

---

## 6. Типы выражений

### 6.1 Арифметические операции

Для `-`, `*`, `/`, `%`:

- оба операнда должны иметь один и тот же числовой тип;
- результат имеет тот же тип.

Для `+`:

- либо оба операнда имеют один и тот же числовой тип, и результат имеет тот же тип;
- либо оба операнда имеют тип `string`, и результат имеет тип `string`.

### 6.2 Логические операции

- `&&`, `||`, `!` применимы только к `bool`;
- результат всегда имеет тип `bool`.

### 6.3 Побитовые операции и сдвиги

- `&`, `|`, `^`, `~`, `<<`, `>>` допустимы только для целочисленных типов.
- Для бинарных побитовых операций и сдвигов типы операндов должны совпадать.
- Результат имеет тот же тип, что и операнд(ы).
- Если величина сдвига отрицательна или не меньше разрядности типа, результатом считается `0`.

### 6.4 Сравнения

- `==` и `!=` допустимы для:
  - числовых типов;
  - `bool`;
  - `char`;
  - `string`.
- `==` и `!=` также допустимы для массивов и структур,
  если их элементы и поля рекурсивно поддерживают equality.
- `<`, `>`, `<=`, `>=` допустимы только для числовых типов.
- В обоих случаях типы операндов должны совпадать.
- Результат любого сравнения имеет тип `bool`.

### 6.5 Конвейерный оператор

Оператор `|>` не создаёт собственного отдельного типа.

- `x |> f` эквивалентно `f(x)`;
- `x |> f(a, b)` эквивалентно `f(x, a, b)`.

Следовательно, тип выражения с `|>` равен типу результата вызова после десугаринга.

### 6.6 Вызов функции

- тип выражения вызова равен типу возврата функции.
- Разрешение вызова может выбирать одну из нескольких перегруженных сигнатур.
- Значения по умолчанию и именованные аргументы не меняют типы параметров, а только способ связывания аргументов с параметрами.

### 6.7 Индексирование массива

Если `a` имеет тип `T[N]`, то выражение `a[i]`:

- требует, чтобы `i` имел целочисленный тип;
- имеет тип `T`.

### 6.8 Доступ к полю структуры

Если `p` имеет тип `S`, а поле `field` объявлено в `S` с типом `T`,
то выражение `p.field` имеет тип `T`.

### 6.9 Выражение `if`

Если условие имеет тип `bool`, а ветви `then` и `else` имеют один и тот же тип `T`,
то выражение

```minic
if (cond) expr1 else expr2
```

имеет тип `T`.

### 6.10 Перегруженные операторы

Если для пользовательского типа объявлена перегрузка оператора, то тип выражения с этим оператором
равен типу возврата выбранной перегруженной функции или метода.

### 6.9 Литерал массива

Литерал массива:

```minic
[e1, e2, ..., eN]
```

имеет тип `T[N]`, если все элементы имеют один и тот же тип `T`.

Если литерал используется в контексте явно ожидаемого типа `T[N]`,
все элементы обязаны совпадать с типом `T`, а число элементов должно быть равно `N`.

### 6.10 Литерал структуры

Литерал структуры:

```minic
Point { x: 1, y: 2 }
```

имеет тип указанной структуры.

Каждое поле:

- должно существовать в структуре;
- должно быть инициализировано ровно один раз;
- должно получить значение строго совпадающего типа.

### 6.11 Присваивание

Если левая часть присваивания имеет тип `T`, то и правая часть обязана иметь тип `T`.

Тип выражения присваивания равен типу левой части.

---

## 7. Явные приведения типов

Единственный допустимый способ преобразовать значение между несовпадающими типами:

```minic
cast<TargetType>(expression)
```

Разрешены:

- любой числовой тип -> любой числовой тип
- любой числовой тип -> `string`
- `bool` -> `string`
- `char` -> `string`

Запрещены:

- `string` -> числовой тип
- `string` -> `bool`
- `struct` -> встроенный тип
- `array` -> встроенный тип
- приведения между структурами
- приведения между массивами

Если приведение не входит в список разрешённых, это семантическая ошибка.

---

## 8. Value semantics

MiniC использует value semantics для всех значений.

Это означает:

- при присваивании скалярных значений копируется само значение;
- при присваивании массива копируется весь массив;
- при присваивании структуры копируется вся структура;
- аргументы функций передаются по значению;
- возвращаемое значение функции концептуально также является значением, а не ссылкой.

Ссылочные типы, указатели и reference semantics в текущей версии языка не поддерживаются.

---

## 9. Типы встроенных функций

В языке доступны следующие встроенные функции:

- `print(T) -> void`, где `T` — любой печатаемый тип:
  - целочисленный;
  - вещественный;
  - `bool`;
  - `char`;
  - `string`
- `input() -> string`
- `len(string) -> int32`
- `len(T[N]) -> int32`
- `exit(int32) -> void`
- `panic(string) -> void`
- `assert(bool) -> void`

Эти функции не являются ключевыми словами и разрешаются семантически как встроенные объявления.

---

## 10. Переполнение и специальные случаи

- целочисленное переполнение не проверяется во время выполнения;
- результат целочисленной арифметики вычисляется по модулю `2^N`,
  где `N` — разрядность соответствующего целочисленного типа;
- поведение `float32` и `float64` при overflow и underflow следует целевой платформе IEEE 754;
- деление на ноль не является ошибкой типов, но является ошибкой времени выполнения.
